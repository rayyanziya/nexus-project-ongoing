🗺️ Nexus — System Architecture Map

This diagram shows how Nexus works — all its components and how they connect. You don't need to be technical to read it.

⭕

What are the circles?

Each circle is a component — a part of the Nexus system with a specific job. For example: "AI Intake", "Billing", "WhatsApp Sender".

↗️

What do the arrows mean?

An arrow from A → B means A talks to B. For example: "AI Intake → Lead Manager" means the chatbot hands off leads to the lead system.

🎨

What do the colours mean?

Each colour is a type of component: Green = business logic, Purple = AI tools, Yellow = data models, Pink = background workers, Grey = external services.

🔍

How do I explore it?

Click any circle to see what it does and what it connects to. Use the left panel to show only one business area at a time.

STEP-BY-STEP: How to read this diagram
1

Start with the AI Intake filter

Click "AI" in the left panel. You'll see just the chatbot, the AI tools it uses, and where leads go after a conversation ends.

2

Follow the arrows

Click any circle. The right panel shows what it connects to. Highlighted arrows show the direct relationships.

3

Zoom in to read labels

Scroll up to zoom in. Labels on arrows explain the relationship (e.g. "calls", "emits", "queues").

4

Press R to reset the view

If you get lost, press the R key to zoom back out and see the full picture.

❓ How to read the Architecture Graph

Quick reference for all audiences.

⭕

Shapes = component types

🟢 Circle = Service (business logic)

🔵 Circle = Controller (API endpoint)

🟡 Diamond = Data model (database table)

🟣 Hexagon = MCP server (AI tool)

⬜ Rectangle = External service

🩷 Ellipse = Background worker
↗️

Arrow labels explained

handles — this endpoint receives that request

calls — this component triggers that one

emits — sends an event when something happens

queues — puts a job in the background queue

uses-mcp — AI agent uses this tool safely

writes / reads — saves or reads from the database
